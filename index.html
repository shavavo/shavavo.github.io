Analytics Dashboard
For a MBA course in the Fuqua Business School, I created three web applications aimed at enhancing the classroom experience by providing data analytics and visualization tools for about 300 students yearly. The three applications- Ratings, Analytics, and Predictive- provide interactive in-class visualizations, and out-of-class analytics and predictive tools involving machine learning and Natural Language Processing.

Interaction. The Ratings app is aimed at enhancing the in class experience. Students perform surveys before and after discussions, and these changes are loaded live in the Ratings application to view. Aditionally, there are tools that randomly select students, show their written justifications, and allows the professor to call on them. Out of class, the students are able to graph any combinations of the data availible from assignments and surveys with the Analytics app, and even create predictive models, create regression models, cluster students, and use Natural Language Processing to analyze justifications of decisions with the Predictive app.
Challenges. Creating this suite of apps provided a complete full stack experience. From database management and job scheduling to user testing and setting up developer/deployment environments, I learned a lot about building, iterating, maintaining, and streamlining a live running app for real users. I gained appreciation for shell scripts, which could automate my deployment pipeline. I also gained appreciation for Docker containers, which further streamlined the deployment process and saved me hours of time. Overall, I learned a lot about a variety of development practices in a full stack environment.

Live Delivery of Automated Fact Checks
Fact Checking
As a part of the Bass Connections Program at Duke, I created a Chrome Extension to deliver live fact checks on the YouTube platform. The extension is at the end of a very sophisticated pipeline that takes a video stream and automatically finds relevant fact checks. Aimed at aiding the 2020 elections, both the fact check extension and pipeline are currently under development.
>

Back
Function. The extension detects if there are fact checks availible for the video being viewed, displays fact checks in real time synchronized with the video, links fact checks to source articles, and provides a history sidebar to view earlier fact checks. Fact checks are synchronized regardless of the player being paused, fast-forwarded, skipped, or speed up. The extension works in both live and non-live settings. The interface is under development and will evolve with user testing.
Challenges. Creating the extension came with plenty of challenges. The main two were synchronizing the production and synthesis of fact checks in a live setting, and creating a experience that challenges the user without alienating. Future work involves the problem of synching different video sources using audio fingerprinting. User testing by Duke Reporter's Lab on a similar problem will also be used to address these issues.

Crowd Sourced Natural Diaster Relief
Foresite
Foresite is platform that enables users to report disaster information at a particular location and then shares that information with other users nearby. These data can be used in real time to identify safe routes through which first responders or evacuees can navigate. Additionally, when a user submits a report, the application provides situation-relevant information to help them stay safe. Foresite placed top 10 at PennApps 2018 for the Resiliency track.

Language Learning Tool
Integrated Language
Integrated Language attempts to integrate language learning in a seamless way by adding vocabulary building to everyday web browsing. In the form of a Google Chrome Extension, Integrated Language reads the HTML of the page a user is viewing and replaces nouns, verbs, and adjectives with a selected language. When hovered over, the words selected turn back into English, and when double clicked, added to a Quizlet set.
